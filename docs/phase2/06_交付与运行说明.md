## RAG 运行模式

| 模式 | 环境变量 | 用途 |
| --- | --- | --- |
| 本地兜底索引 | `PHASE2_VECTOR_BACKEND=local` 或不设置 | 不启动 Milvus 也能演示业务闭环 |
| 真实 Milvus RAG | `PHASE2_VECTOR_BACKEND=milvus` | 使用 LangChain + Milvus + BGE + BM25 + Rerank |

真实 RAG 模式使用：

- `backend/app/rag/milvus_store.py`
- `langchain_milvus.Milvus`
- `langchain_huggingface.HuggingFaceEmbeddings`
- `BM25BuiltInFunction`
- `sentence_transformers.CrossEncoder`

## Windows 一键启动

当前项目根目录已经提供 Windows 启动脚本。你在 PowerShell / CMD 中不要运行 `.sh`，请运行：

```powershell
.\start_project.bat
```

如果提示依赖缺失，首次运行请执行：

```powershell
.\start_project.bat -InstallDeps
```

默认 local 模式只安装 `backend/requirements-runtime.txt` 中的轻量后端运行依赖。只有使用真实 Milvus RAG 时，才安装包含 LangChain、Milvus、BGE、Reranker 的完整 `backend/requirements.txt`。

Windows 一键脚本默认设置 `DB_BACKEND=sqlite`，数据库文件为 `backend/storage/slc_phase2.db`，因此 local MVP 不再强制依赖本机 MySQL。需要使用 MySQL 时可显式设置 `DB_BACKEND=mysql`。

停止项目：

```powershell
.\stop_project.bat
```

如果要使用真实 Milvus RAG：

```powershell
docker compose -f docker-compose.milvus.yml up -d
.\start_project.bat -InstallDeps -VectorBackend milvus
```

说明：`scripts/start_project.sh` 是 Bash 脚本，只适合 Git Bash、WSL 或 Linux/macOS 终端。
