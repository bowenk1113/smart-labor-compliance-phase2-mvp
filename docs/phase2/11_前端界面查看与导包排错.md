# 前端界面查看与导包排错

本文用于解决“前端打不开”“导包报错”“不知道访问哪个地址”的问题。

## 1. 先确认使用 RAG conda 环境

本项目后端依赖安装在：

```text
E:\anaconda\envs\RAG
```

启动脚本会自动使用：

```text
E:\anaconda\envs\RAG\python.exe
```

脚本已设置：

```text
PYTHONNOUSERSITE=1
```

这表示后端启动时只使用 RAG conda 环境里的包，不混用 `C:\Users\...\AppData\Roaming\Python312\site-packages` 里的用户级 Python 包。

## 7. 常见导包错误

### 7.1 `ModuleNotFoundError: No module named 'jose'`

原因：RAG conda 环境缺少 `python-jose`。

修复：

```powershell
$env:PYTHONNOUSERSITE="1"
E:\anaconda\envs\RAG\python.exe -m pip install --no-user -r backend\requirements-runtime.txt
```

### 7.2 `ModuleNotFoundError: No module named 'typing_extensions'`

原因：RAG conda 环境没有安装该依赖，或 Python 混用了用户级包路径。

修复：

```powershell
$env:PYTHONNOUSERSITE="1"
E:\anaconda\envs\RAG\python.exe -m pip install --no-user typing_extensions==4.15.0
```

### 7.3 前端 `vite` 找不到

原因：`frontend/node_modules` 不存在。

修复：

```powershell
cd frontend
npm install
```

或从项目根目录执行：

```powershell
.\start_project.bat -InstallDeps
```

### 7.4 PowerShell 中文显示乱码

如果 `Invoke-RestMethod` 输出中文变成乱码，但浏览器页面正常中文，这是 PowerShell 控制台编码显示问题，不代表后端返回错误。

建议用浏览器查看：

```text
http://localhost:3000
```
