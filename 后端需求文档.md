# 企业用工与社保合规智能平台 - 后端需求文档

## 一、文档概述

本文档描述企业用工与社保合规智能平台的后端需求，基于 Dify + Python + FastAPI + MySQL 技术架构，首期 MVP 聚焦陕西地区用工与社保合规场景。

## 二、技术架构

### 2.1 总体架构

```
用户端 Web 页面  ──┐
                   ├──> Python FastAPI 后端 ──> Dify API / Chatflow
管理后台 Web 页面 ──┘         │
                              ├──> MySQL 数据库
                                    ├──> 问答记录表
                                    ├──> 反馈记录表
                                    ├──> FAQ数据表
                                    └──> 来源目录表
```

### 2.2 技术栈

- Python 3.x
- FastAPI 框架
- MySQL 数据库
- Dify API 调用
- 可选：Nginx / Docker

### 2.3 项目结构

参考现有项目：student-management-backend

```
student-management-backend/
├── app.py              # 主应用入口
├── database.py         # 数据库连接
├── security.py         # 安全相关
├── i18n.py             # 国际化
├── check_db.py         # 数据库检查
├── requirements.txt    # 依赖
└── README.md
```

## 三、核心 API 接口需求

### 3.1 问答题接口

#### 3.1.1 聊天接口

```
POST /api/chat

请求参数：
Content-Type: application/json

{
  "question": "用户问题文本",
  "user_id": "用户ID（可选）",
  "session_id": "会话ID（可选）"
}

响应成功 (200)：
{
  "success": true,
  "data": {
    "answer": "系统回答内容",
    "question_id": "问题记录ID",
    "sources": [
      {
        "title": "来源标题",
        "url": "来源链接",
        "date": "发布日期"
      }
    ],
    "related_tasks": [
      {
        "title": "办事事项名称",
        "path": "办事路径说明",
        "contact": "联系方式"
      }
    ],
    "response_time": 响应时间（毫秒）
  }
}

响应失败 (500)：
{
  "success": false,
  "error": "错误信息"
}
```

#### 3.1.2 Dify 调用服务

- 调用 Dify Chatflow API
- 处理知识检索
- 处理语义召回
- 处理问答生成
- Prompt 约束管理

### 3.2 历史记录接口

#### 3.2.1 获取历史记录

```
GET /api/history

请求参数：
- user_id: 用户ID（必填）
- page: 页码（默认1）
- page_size: 每页条数（默认10）

响应成功 (200)：
{
  "success": true,
  "data": {
    "list": [
      {
        "id": 记录ID,
        "question": "问题内容",
        "answer": "回答内容",
        "created_at": "创建时间",
        "is_feedback": 是否已反馈
      }
    ],
    "total": 总数,
    "page": 当前页码,
    "page_size": 每页条数
  }
}
```

#### 3.2.2 清空历史记录

```
DELETE /api/history

请求参数：
- user_id: 用户ID（必填）

响应成功 (200)：
{
  "success": true
}
```

### 3.3 反馈接口

#### 3.3.1 提交反馈

```

请求参数：
Content-Type: application/json

{
  "question_id": "问题记录ID",
  "is_helpful": true/false,
  "remark": "反馈备注（可选）",

}

响应成功 (200)：
{

  "data": {
    "feedback_id": "反馈记录ID"
  }

```

## 四、后端管理 API 接口需求

### 4.1 认证接口

#### 4.1.1 管理员登录

```
POST /api/admin/login

请求参数：
Content-Type: application/json

{
  "username": "用户名",
  "password": "密码"
}

响应成功 (200)：
{
  "success": true,
  "data": {
    "token": "JWT令牌",
    "admin_id": "管理员ID",
    "username": "用户名",
    "role": "角色"
  }
}

响应失败 (401)：
{
  "success": false,
  "error": "用户名或密码错误"
}
```

#### 4.1.2 验证令牌

```
GET /api/admin/verify-token

请求头：
Authorization: Bearer <token>

响应成功 (200)：
{
  "success": true,
  "data": {
    "admin_id": "管理员ID",
    "username": "用户名"
  }
}
```

```
  "data": {
    "list": [
      {
        "id": 日志ID,
        "question": "用户问题",

    "page": 当前页码,
    "page_size": 每页条数
  }
}
```

## 五、数据库设计需求

### 5.1 数据表结构

#### 5.1.1 问答记录表 (chat_logs)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| user_id | VARCHAR(64) | 用户ID |
| session_id | VARCHAR(64) | 会话ID |
| question | TEXT | 用户问题 |
| answer | TEXT | 系统回答 |
| sources | JSON | 来源列表 |
| related_tasks | JSON | 办事路径 |
| response_time | INT | 响应时间（毫秒） |
| status | VARCHAR(20) | 状态（success/failed） |
| created_at | DATETIME | 创建时间 |

#### 5.1.2 反馈记录表 (feedbacks)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| question_id | INT | 问答记录ID |
| user_id | VARCHAR(64) | 用户ID |
| is_helpful | BOOLEAN | 是否有帮助 |
| remark | TEXT | 备注 |
| status | VARCHAR(20) | 处理状态 |
| created_at | DATETIME | 创建时间 |

#### 5.1.3 FAQ 表 (faqs)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| question | VARCHAR(500) | 问题 |
| answer | TEXT | 答案 |
| category | VARCHAR(50) | 分类 |
| keywords | JSON | 关键词 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 5.1.4 来源目录表 (sources)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| title | VARCHAR(200) | 文档标题 |
| url | VARCHAR(500) | 文档链接 |
| doc_type | VARCHAR(20) | 文档类型 |
| region | VARCHAR(50) | 适用地区 |
| publish_date | DATE | 发布日期 |
| description | TEXT | 描述 |
| created_at | DATETIME | 创建时间 |

#### 5.1.5 知识包表 (knowledge_packages)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| name | VARCHAR(100) | 知识包名称 |
| region | VARCHAR(50) | 适用地区 |
| description | TEXT | 描述 |
| status | VARCHAR(20) | 状态 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

#### 5.1.6 管理员表 (admins)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| username | VARCHAR(50) | 用户名 |
| password_hash | VARCHAR(200) | 密码哈希 |
| role | VARCHAR(20) | 角色 |
| created_at | DATETIME | 创建时间 |

### 5.2 索引设计

- chat_logs: user_id, created_at
- feedbacks: question_id, created_at
- faqs: category
- sources: doc_type, region

## 六、安全需求

### 6.1 认证与授权

- 管理员登录使用 JWT 令牌
- Token 过期时间：24小时
- 密码使用哈希存储（bcrypt）

### 6.2 接口安全

- 管理员接口需要验证 Token
- 防止 SQL 注入
- 防止 XSS 攻击
- 请求频率限制（可选）

### 6.3 数据安全

- 用户敏感信息脱敏
- 日志记录审计

## 七、非功能性需求

### 7.1 性能要求
